# Configuration

ssh-web is configured via `config.yaml` in the application root directory. The file is auto-generated on first startup if it doesn't exist.

## Load Process

1. If `config.yaml` exists: parse YAML, fill empty fields with defaults/env vars, use the result
2. If `config.yaml` does not exist: create it with default values (env vars override defaults), then load
3. Auto-generated secrets (`password_hash`, `totp_secret`, `encryption_key`) are created once and preserved in the file

## Reference

### `server`

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `port` | int | `8080` | HTTP listen port |
| `tls_cert` | string | `""` | Path to TLS certificate file (empty = no TLS) |
| `tls_key` | string | `""` | Path to TLS key file (empty = no TLS) |
| `base_path` | string | `""` | URL path prefix for all routes, e.g. `/ssh-web` |

Example:

```yaml
server:
  port: 8080
  base_path: "/ssh-web"
```

### `auth`

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `username` | string | `admin` | Web UI login username |
| `password_hash` | string | auto | bcrypt hash of the login password |
| `totp_secret` | string | auto | Base32-encoded TOTP secret for 2FA |
| `auth_enabled` | bool | `true` | Enable/disable login page and auth checks |

`password_hash` is a bcrypt hash generated by the Go standard library (`golang.org/x/crypto/bcrypt`). The default password is `changeme`.

`totp_secret` follows RFC 4648 Base32 (uppercase A-Z, digits 2-7). Scan this with an authenticator app (Google Authenticator, Authy, etc.) to get TOTP codes.

`auth_enabled` controls whether the web UI requires authentication. When set to `false`, the login and TOTP pages are bypassed — visitors go directly to the terminal interface. The SSH connection still uses its own credentials from `default_host`.

Example:

```yaml
auth:
  username: admin
  password_hash: "$2a$10$..."
  totp_secret: "LRDIGKMVKBFQJ57ZVKGFCCX7LAPXG7IQ"
  auth_enabled: true
```

### `encryption_key`

| Type | Default | Description |
|------|---------|-------------|
| string | auto | Base64-encoded 32-byte key for AES-256 encryption |

Used to encrypt `default_host.password_encrypted` so SSH passwords are never stored in plain text.

Generate an encrypted password:

```bash
./ssh-web encrypt-password "your-ssh-password"
```

### `default_host`

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `host` | string | `127.0.0.1` | Target SSH server hostname or IP |
| `port` | int | `22` | Target SSH server port |
| `username` | string | `root` | SSH login username |
| `auth_method` | string | `password` | Authentication method: `password` or `private_key` |
| `password_encrypted` | string | `""` | Encrypted SSH password (use the `encrypt-password` tool) |
| `private_key_path` | string | `""` | Path to SSH private key file |
| `host_key_check` | bool | `true` | Verify SSH host key (`~/.ssh_web/known_hosts`) |

Example:

```yaml
default_host:
  host: 192.168.1.100
  port: 22
  username: pi
  auth_method: password
  password_encrypted: "Bsl75Ih97lFXrqITFbRMpRKncz1wzGazDzj3wbm+H+CywayvDzbq+qM="
  host_key_check: true
```

## Full Example

```yaml
server:
  port: 8080
  tls_cert: "/etc/ssl/certs/server.crt"
  tls_key: "/etc/ssl/private/server.key"
  base_path: ""
auth:
  username: admin
  password_hash: "$2a$10$1XOhA1otCuUAPI/.zTubYuDet6y4GSovjq42Ad9c4dR9BLBhPH9Iu"
  totp_secret: "LRDIGKMVKBFQJ57ZVKGFCCX7LAPXG7IQ"
  auth_enabled: true
encryption_key: "UlBDRzlJUkVHUUNvY21Yb3UzMHc5SWFGUjg3d21rZS0="
default_host:
  host: 192.168.1.100
  port: 22
  username: pi
  auth_method: password
  password_encrypted: "Bsl75Ih97lFXrqITFbRMpRKncz1wzGazDzj3wbm+H+CywayvDzbq+qM="
  private_key_path: ""
  host_key_check: true
```

## Environment Variable Overrides

All fields can be set via `SSH_WEB_*` environment variables. See [env.md](env.md) for the full reference.
